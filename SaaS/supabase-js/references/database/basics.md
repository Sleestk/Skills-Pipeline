# Database Basics

## Tables

### Creating tables
```sql
create table movies (
  id bigint generated by default as identity primary key,
  name text,
  description text
);
```
Use lowercase and underscores for table names (e.g., `table_name`, not `Table Name`).

### Data types
Common types: `bigint`/`int8`, `integer`/`int4`, `text`, `boolean`/`bool`, `uuid`, `timestamptz`, `jsonb`, `numeric`/`decimal`, `real`/`float4`, `double precision`/`float8`, `date`, `time`, `bytea`.

### Primary keys
```sql
-- Auto-incrementing (recommended)
create table movies (
  id bigint generated always as identity primary key
);

-- Or allow manual inserts
create table movies (
  id bigint generated by default as identity primary key
);
```

### Loading data
```sql
insert into movies (name, description)
values
  ('The Empire Strikes Back', 'Luke begins Jedi training.'),
  ('Return of the Jedi', 'Rebels dispatch to Endor.');
```

```js
const { data, error } = await supabase.from('movies').insert([
  { name: 'The Empire Strikes Back', description: 'Luke begins Jedi training.' },
  { name: 'Return of the Jedi', description: 'Rebels dispatch to Endor.' },
])
```

### Bulk loading (COPY)
```bash
psql -h DATABASE_URL -p 5432 -d postgres -U postgres \
  -c "\COPY movies FROM './movies.csv' WITH DELIMITER ',' CSV HEADER"
```

### Foreign keys (joining tables)
```sql
create table categories (
  id bigint generated always as identity primary key,
  name text
);

alter table movies
  add column category_id bigint references categories;
```

### Many-to-many (join table)
```sql
create table performances (
  id bigint generated by default as identity primary key,
  movie_id bigint not null references movies,
  actor_id bigint not null references actors
);
```

### Schemas
```sql
create schema private;
create table private.salaries (
  id bigint generated by default as identity primary key,
  salary bigint not null,
  actor_id bigint not null references public.actors
);
```

## Views

```sql
create view transcripts as
  select students.name, courses.title, grades.result
  from grades
  left join students on grades.student_id = students.id
  left join courses on grades.course_id = courses.id;

grant all on table transcripts to authenticated;
```

### Security invoker (Postgres 15+)
Makes the view respect RLS of the calling role:
```sql
create view my_view with(security_invoker=true) as (
  select * from my_table
);
```

### Materialized views
```sql
create materialized view transcripts as
  select students.name, courses.title, grades.result
  from grades
  left join students on grades.student_id = students.id
  left join courses on grades.course_id = courses.id;

-- Refresh manually
refresh materialized view transcripts;
```

## Arrays

```sql
create table arraytest (
  id integer not null,
  textarray text array
);

INSERT INTO arraytest (id, textarray) VALUES (1, ARRAY['Harry', 'Larry', 'Moe']);
```

```js
const { data, error } = await supabase
  .from('arraytest')
  .insert([{ id: 2, textarray: ['one', 'two', 'three'] }])
```

Postgres uses 1-based indexing: `textarray[1]` is the first item.

## Indexes

```sql
-- Basic B-Tree index
create index idx_persons_age on persons (age);

-- Concurrent (no write lock)
create index concurrently idx_persons_age on persons (age);

-- Partial index
create index idx_living_persons_age on persons (age)
where deceased is false;

-- Descending order
create index idx_persons_age_desc on persons (age desc nulls last);

-- Reindex
reindex index concurrently idx_persons_age;
```

**Always add indexes on columns used in RLS policies.**

## Joins

### One-to-many
```js
const { data, error } = await supabase.from('orchestral_sections').select(`
  id, name,
  instruments ( id, name )
`)
```

### TypeScript types for joins
```ts
import { QueryData } from '@supabase/supabase-js'

const query = supabase.from('orchestral_sections').select(`
  id, name, instruments ( id, name )
`)
type Result = QueryData<typeof query>
```

### Inner join (`!inner`)
Filters out parent rows with no match:
```js
const { data, error } = await supabase
  .from('orchestral_sections')
  .select(`id, name, instruments!inner ( id, name )`)
  .eq('instruments.name', 'flute')
```

### Many-to-many
Auto-detected through join tables:
```js
const { data, error } = await supabase.from('teams').select(`
  id, team_name,
  users ( id, name )
`)
```

### Multiple foreign keys (disambiguate with `!foreign_key`)
```js
const { data, error } = await supabase.from('shifts').select(`
  *,
  start_scan:scans!scan_id_start ( id, badge_scan_time ),
  end_scan:scans!scan_id_end ( id, badge_scan_time )
`)
```

### Join syntax reference
| Syntax | Meaning |
|---|---|
| `alias:relation(columns)` | Rename the embedded relation |
| `relation!inner(columns)` | Inner join behavior |
| `relation!foreign_key(columns)` | Disambiguate which FK to use |

## JSON / JSONB

Use `jsonb` (not `json`) for almost all cases — faster to process.

```sql
create table books (
  id serial primary key,
  title text,
  author text,
  metadata jsonb
);
```

### Inserting JSON
```js
const { data, error } = await supabase.from('books').insert([
  {
    title: 'Green Eggs and Ham',
    author: 'Dr. Seuss',
    metadata: { description: 'Unusual food.', price: 7.49, ages: [4, 8] },
  },
])
```

### Querying JSON
SQL operators: `->` returns jsonb, `->>` returns text.
```sql
select title, metadata ->> 'description' as description, metadata -> 'price' as price
from books;
```

```js
const { data, error } = await supabase.from('books').select(`
  title,
  description: metadata->>description,
  price: metadata->price,
  low_age: metadata->ages->0,
  high_age: metadata->ages->1
`)
```

### Validating JSON with pg_jsonschema
```sql
alter table customers
add constraint check_metadata check (
  json_matches_schema('{"type":"object","properties":{"tags":{"type":"array","items":{"type":"string","maxLength":16}}}}', metadata)
);
```
